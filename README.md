# Таблица уровней стажёров LM.inc

Статический сайт с таблицей рейтинга (место / номер пропуска / баллы / уровень),
которая автоматически обновляется данными с
`https://lm-inc-levels-knyazhna.amvera.io`.

## Как это работает

- `index.html` — сама страница с таблицей. Читает `data.json` из этого же
  репозитория (без CORS-проблем, т.к. файл лежит рядом) и рендерит таблицу,
  отсортированную по баллам.
- `data.json` — снимок данных (сотрудники, серийные номера, баллы, уровень).
- `scripts/update-data.mjs` — Node-скрипт, который скачивает страницу-источник,
  вытаскивает из неё встроенный список сотрудников и пересобирает `data.json`.
- `.github/workflows/update-data.yml` — GitHub Action, который запускает этот
  скрипт по расписанию (каждые 15 минут) и коммитит `data.json`, если данные
  изменились.

Таким образом таблица обновляется сама, без вашего участия: раз в 15 минут
Action проверяет источник → если баллы/уровни изменились → обновляет
`data.json` в репозитории → сайт на GitHub Pages подхватывает новый файл.

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub (например, `lm-inc-levels`).
2. Загрузите в него все файлы из этой папки, сохранив структуру:
   ```
   index.html
   data.json
   README.md
   scripts/update-data.mjs
   .github/workflows/update-data.yml
   ```
   Проще всего — через `git`:
   ```bash
   cd lm-inc-levels
   git init
   git add .
   git commit -m "Первый коммит"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/lm-inc-levels.git
   git push -u origin main
   ```
3. В репозитории откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите:
   - Source: **Deploy from a branch**
   - Branch: **main**, папка **/ (root)**
5. Сохраните — через минуту-две сайт будет доступен по адресу вида
   `https://ВАШ_ЛОГИН.github.io/lm-inc-levels/`.
6. Проверьте вкладку **Actions** — там должен появиться workflow
   «Обновление данных LM.inc». Он выполнится автоматически по расписанию,
   но можно сразу запустить вручную кнопкой **Run workflow**, чтобы обновить
   данные сразу после публикации.

## Важно про права Action на пуш в репозиторий

Если Action не сможет запушить обновлённый `data.json` (ошибка permissions),
зайдите в **Settings → Actions → General → Workflow permissions** и
выберите **Read and write permissions**, затем сохраните.

## Локальная проверка

```bash
node scripts/update-data.mjs   # обновит data.json из реального источника
python3 -m http.server 8000    # или любой другой статический сервер
# откройте http://localhost:8000
```

## Если сайт-источник изменит вёрстку

Скрипт `scripts/update-data.mjs` ищет в HTML-странице подстроку
`"employees":[...]` (это встроенные Next.js данные). Если разработчики
источника изменят структуру страницы и поиск перестанет находить эти данные,
Action просто завершится с ошибкой в логах (`data.json` при этом не
перезатирается — сайт продолжит показывать последние актуальные данные),
и вам нужно будет посмотреть свежий HTML страницы и поправить скрипт.
