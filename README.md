# Таблица уровней стажёров LM.inc

Статический сайт с таблицей рейтинга (место / номер пропуска / баллы / уровень),
которая автоматически обновляется данными с
`https://lm-inc-levels-knyazhna.amvera.io`, шлёт уведомления об изменениях
баллов в Telegram и хранит архив прошлых версий таблицы.

## Как это работает

- `index.html` — сама страница с таблицей. Читает `data.json` из этого же
  репозитория и рендерит таблицу, отсортированную по баллам. Рядом с полем
  поиска — выпадающий список для просмотра архивных версий таблицы.
- `data.json` — текущий снимок данных (сотрудники, серийные номера, баллы,
  уровень).
- `history/` — архив предыдущих версий `data.json` (по одному файлу на
  каждое обнаруженное изменение) и `history/index.json` со списком этих
  файлов для выпадающего списка на сайте.
- `scripts/update-data.mjs` — Node-скрипт, который скачивает страницу-источник,
  вытаскивает из неё встроенный список сотрудников, сравнивает с прошлым
  снимком, сохраняет архив, отправляет уведомление в Telegram (если настроено)
  и пересобирает `data.json`.
- `.github/workflows/update-data.yml` — GitHub Action, который запускает этот
  скрипт по расписанию (каждые 15 минут) и коммитит изменения `data.json` и
  `history/`, если данные изменились.

Таким образом таблица обновляется сама, без вашего участия: раз в 15 минут
Action проверяет источник → если баллы/уровни изменились → сохраняет старую
версию в архив → отправляет уведомление в Telegram → обновляет `data.json` →
сайт на GitHub Pages подхватывает новый файл.

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub (например, `lm-inc-levels`).
2. Загрузите в него все файлы из этой папки, сохранив структуру:
   ```
   index.html
   data.json
   README.md
   history/index.json
   scripts/update-data.mjs
   .github/workflows/update-data.yml
   ```
   Проще всего — через `git`:
   ```bash
   cd lm-inc-levels
   git init
   git add .
   git commit -m "Первый коммит"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/lm-inc-levels.git
   git push -u origin main
   ```
3. В репозитории откройте **Settings → Pages**.
4. В разделе **Build and deployment** выберите:
   - Source: **Deploy from a branch**
   - Branch: **main**, папка **/ (root)**
5. Сохраните — через минуту-две сайт будет доступен по адресу вида
   `https://ВАШ_ЛОГИН.github.io/lm-inc-levels/`.
6. Проверьте вкладку **Actions** — там должен появиться workflow
   «Обновление данных LM.inc». Он выполнится автоматически по расписанию,
   но можно сразу запустить вручную кнопкой **Run workflow**, чтобы обновить
   данные сразу после публикации.

## Важно про права Action на пуш в репозиторий

Если Action не сможет запушить обновлённый `data.json` (ошибка permissions),
зайдите в **Settings → Actions → General → Workflow permissions** и
выберите **Read and write permissions**, затем сохраните.

## Уведомления в Telegram при изменении баллов

Каждый раз, когда Action обнаруживает изменение баллов у кого-то из
стажёров, он отправляет сообщение в Telegram-бота — с номером пропуска,
старым и новым количеством баллов и разницей. Например:

```
Изменения баллов LM.inc (2):
№00007: 79.6 → 82.1 (+2.5)
№00012: 74.3 → 71.0 (-3.3)
```

### Настройка (один раз)

1. В Telegram напишите **@BotFather** → `/newbot` → придумайте имя боту →
   получите **токен** (строка вида `123456789:ABCdefGhIJKlmNoPQRsTUVwxyz`).
2. Напишите что-нибудь новому боту (любое сообщение) — иначе он не сможет
   писать вам первым.
3. Узнайте свой **chat_id**: откройте в браузере (подставив свой токен)
   `https://api.telegram.org/botТОКЕН/getUpdates`, отправьте боту ещё одно
   сообщение и обновите страницу — в ответе найдите `"chat":{"id":ЧИСЛО`.
   Это число и есть chat_id (может быть отрицательным, если это группа —
   это нормально, просто скопируйте как есть).
4. В репозитории на GitHub: **Settings → Secrets and variables → Actions →
   New repository secret**. Добавьте два секрета:
   - `TELEGRAM_BOT_TOKEN` — токен бота
   - `TELEGRAM_CHAT_ID` — chat_id из шага 3

Готово — при следующем изменении баллов Action сам отправит сообщение.
Если секреты не заданы, уведомления просто не отправляются — остальной
сайт продолжает работать как обычно.

## Архив старых таблиц

Перед каждой перезаписью `data.json` Action сохраняет предыдущую версию
таблицы в папку `history/` (файл с меткой времени) и добавляет запись в
`history/index.json`. На главной странице сайта появляется выпадающий
список рядом с полем поиска — выбрав в нём дату, можно посмотреть, как
выглядела таблица до изменений. Кнопка «Вернуться к текущей» (или выбор
пункта «Текущая таблица») возвращает к живым данным.

Ничего дополнительно настраивать не нужно — это работает сразу после
первого изменения баллов.

## Локальная проверка

```bash
node scripts/update-data.mjs   # обновит data.json из реального источника
python3 -m http.server 8000    # или любой другой статический сервер
# откройте http://localhost:8000
```

## Если сайт-источник изменит вёрстку

Скрипт `scripts/update-data.mjs` ищет в HTML-странице подстроку
`"employees":[...]` (это встроенные Next.js данные). Если разработчики
источника изменят структуру страницы и поиск перестанет находить эти данные,
Action просто завершится с ошибкой в логах (`data.json` при этом не
перезатирается — сайт продолжит показывать последние актуальные данные),
и вам нужно будет посмотреть свежий HTML страницы и поправить скрипт.
